# Seismic Pattern Modeler

Seismic Pattern Modeler is a Python workflow for analyzing California earthquake catalogs and generating outputs for:

- Gutenberg–Richter magnitude-frequency fitting
- Omori aftershock sequence validation
- Aftershock-rate prediction with XGBoost
- Interactive seismic dashboard mapping

## Project Structure

- `main.py` — orchestrates the full workflow
- `src/pipeline.py` — data fetch and cleaning
- `src/gutenberg_richter.py` — Gutenberg–Richter analysis
- `src/omori.py` — Omori validation
- `src/model.py` — feature engineering and model training
- `src/dashboard.py` — dashboard/map generation
- `outputs/` — generated plots, CSV, and HTML outputs

## Requirements

- Python 3.10+
- Dependencies listed in `requirements.txt`

Install dependencies:

```bash
pip install -r requirements.txt
```

## Running the Workflow

From the repository root, run:

```bash
python main.py
```

The pipeline will create `data/` and `outputs/` directories as needed, process the catalog, run analyses, train the model, and generate visual artifacts.

## Expected Outputs

Typical artifacts include:

- `outputs/gutenberg_richter.png`
- `outputs/omori_fits.png`
- `outputs/omori_results.csv`
- `outputs/seismic_map.html`
- `feature_importance.png`, `model_residuals.png`, and model files generated by training
